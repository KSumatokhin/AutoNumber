# AutoNumber

`AutoNumber` — плагин для BricsCAD, который последовательно заполняет выбранный атрибут одинаковых блоков. Плагин обрабатывает пространства листов напрямую через базу чертежа: для массовой нумерации не требуется открывать или последовательно активировать каждый лист.

## Возможности

- выбор блока-образца непосредственно в чертеже;
- поддержка обычных и динамических блоков;
- автоматическое определение доступных атрибутов образца;
- нумерация всех подходящих блоков на листах или только выбранных объектов;
- необязательная обработка пространства модели;
- начальный номер, шаг, префикс и суффикс;
- обработка листов в порядке вкладок;
- сортировка блоков сверху вниз и слева направо;
- изменение всех атрибутов в одной транзакции без переключения листов.

## Требования

- Windows;
- BricsCAD V21 x64;
- .NET Framework 4.7.2;
- Visual Studio с поддержкой проектов .NET Framework — только для самостоятельной сборки.

Проект ссылается на библиотеки BricsCAD из стандартной папки русской версии V21:

```text
C:\Program Files\Bricsys\BricsCAD V21 ru_RU\BrxMgd.dll
C:\Program Files\Bricsys\BricsCAD V21 ru_RU\TD_Mgd.dll
```

Если BricsCAD установлен в другом каталоге, измените `HintPath` соответствующих ссылок в `AutoNumber.csproj`.

## Сборка

Откройте `AutoNumber.slnx` в Visual Studio и соберите конфигурацию `Debug`.

Результат сборки:

```text
bin\Debug\AutoNumber.dll
```

Также проект можно собрать из Developer PowerShell for Visual Studio:

```powershell
msbuild AutoNumber.csproj /t:Build /p:Configuration=Debug /p:Platform=AnyCPU
```

Несмотря на имя платформы `AnyCPU`, Debug-конфигурация проекта создаёт x64-сборку для BricsCAD.

## Установка и запуск

1. Запустите BricsCAD.
2. Выполните команду `NETLOAD`.
3. Выберите собранный файл `AutoNumber.dll`.
4. Запустите команду `AutoNumber`.

Если Windows блокирует DLL, откройте свойства файла в Проводнике, установите флажок «Разблокировать» и повторите `NETLOAD`.

## Использование

1. После запуска `AutoNumber` выберите блок-образец.
2. В окне настроек выберите тег атрибута. Если у блока есть атрибут `NUM`, он выбирается автоматически.
3. Укажите начальный номер и шаг.
4. При необходимости задайте префикс и суффикс. Например, префикс `B1-` сформирует значения `B1-1`, `B1-2`, `B1-3`.
5. Выберите область обработки:
   - **Все листы** — подходящие блоки будут найдены во всех пространствах листов;
   - **Выбрать объекты после закрытия окна** — BricsCAD предложит выбрать блоки вручную.
6. Нажмите **Нумеровать**.

Обрабатываются только блоки с тем же эффективным именем, что и у образца, и содержащие выбранный атрибут. Регистр имени тега не учитывается.

## Порядок нумерации

В режиме обработки всех листов плагин использует следующий порядок:

1. листы — по порядку вкладок BricsCAD;
2. блоки на листе — сверху вниз;
3. блоки в одной строке — слева направо.

Точки с разницей координаты Y не более `0.1` единицы чертежа считаются одной строкой.

## Важные особенности

- Плагин изменяет только текст выбранного атрибута.
- Геометрия, точка вставки и поворот блока или атрибута не изменяются.
- Счётчик увеличивается только после успешного изменения подходящего атрибута.
- В режиме «Все листы» пространство модели не обрабатывается, пока не установлен отдельный флажок.
- Перед массовой обработкой рекомендуется сохранить резервную копию DWG.

## Структура проекта

- `AutoNumberCommand.cs` — команда BricsCAD, поиск, сортировка и изменение блоков;
- `MainWindow.xaml` — окно параметров нумерации;
- `ViewModels/NumberingSettingsViewModel.cs` — состояние окна;
- `ResourceDictionary.xaml` — стили интерфейса;
- `Models/Enums.cs` — режимы обработки.

## Происхождение

Проект развивает прежний сценарий автоматической нумерации атрибута `NUM`, реализованный на AutoLISP. Версия на C# предназначена прежде всего для быстрой пакетной обработки большого количества листов без их последовательного открытия.
